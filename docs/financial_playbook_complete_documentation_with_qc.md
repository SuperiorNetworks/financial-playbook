# Financial Playbook - Complete Documentation with QC Numbers

This document provides a comprehensive overview of the Financial Playbook application, including wireframes with QC numbering, technical specifications, and integration details.

**QC Numbering System:** Every UI element is assigned a unique 3-digit number in brackets (e.g., [101]) for easy reference during development and communication. These numbers can be toggled off for production release.

---

## Application Overview

**AI-Driven Financial Playbook Website** - A single-user web application that provides comprehensive financial scenario planning, cash flow monitoring, and automated risk analysis with real-time QuickBooks integration.

## System Components

### 1. Authentication System
- Google OAuth 2.0 integration
- Session management with JWT
- Secure cookie handling
- Single-user architecture

### 2. QuickBooks Online Integration
- OAuth 2.0 connection flow
- Real-time data synchronization via webhooks
- Automatic sync on login
- Manual sync trigger
- Configurable background sync frequency (hourly, 4-hour, 8-hour, daily)
- **All financial data (balances, transactions, bills, invoices) sourced from QuickBooks API**
- **READ-ONLY data policy:** QuickBooks data cannot be edited or deleted within this application
- "View in QuickBooks" links for all QuickBooks records to make changes at the source

### 3. AI Strategy Builder
- Document upload (PDF, PNG, JPG)
- OCR and text extraction
- AI-powered data extraction
- **Mandatory math validation layer** - All AI calculations verified by traditional programming
- Transaction categorization
- Recurring bill identification
- Strategy recommendation engine

### 4. Logic Simulator & Report Generator
- Visual logic editor for conditional rules
- Day-by-day simulation engine
- Risk flagging system
- **AI Math Validation** - All report calculations double-checked
- AI-powered root cause analysis
- AI-generated solution recommendations
- Multi-format export (PDF, Word, Markdown, CSV)

### 5. Automation System
- Task status management (Manual, Automated)
- Price variance alerts (default 3%)
- Email notification engine
- Configurable alert thresholds

### 6. Audit Trail
- Immutable logging of all actions
- Before-and-after snapshots
- Filterable by date and action type
- CSV export for compliance

### 7. UI Quality Control System
- Unique 3-digit numbering for all UI elements
- Development mode toggle
- Easy removal for production launch

## Technical Architecture

### Frontend
- React 19 with Next.js
- Tailwind CSS 4
- tRPC 11 for type-safe API
- Chart.js for visualizations

### Backend
- Express 4
- tRPC 11 procedures
- MySQL/TiDB with Drizzle ORM
- Serverless functions on Vercel

### External Integrations & Connectors

**Core Integrations:**
- **QuickBooks Online API** (OAuth 2.0) - Primary financial data source for accounts, transactions, bills, and invoices
- **Google OAuth 2.0** - User authentication and identity management
- **AI/LLM API** (OpenAI GPT-4 or Grok) - Document analysis, data extraction, strategy recommendations, and report generation (with mandatory math validation)
- **Email Service** (SendGrid, AWS SES, or similar) - Notification delivery for alerts and automation
- **Cloud Storage** (S3 or similar) - Secure storage for uploaded financial documents

**Future Connectors:**
- **Plaid API** - Additional bank account connections beyond QuickBooks (if needed)
- **Stripe API** - Payment processing integration (if monetizing the platform)

**Data Handling Policy:**
- QuickBooks data is **READ-ONLY** - all edits must be made in QuickBooks Online
- User-created scenarios, rules, and playbooks are fully editable within the application
- All uploaded documents are stored securely with encryption at rest

### Security
- HTTPS enforcement
- CSRF protection
- XSS prevention
- SQL injection prevention via ORM
- Encrypted storage for OAuth tokens
- Rate limiting on API endpoints
